# 贡献指南
感谢您关注并参与 notary-sop 项目！无论是代码优化、流程完善、文档修正，我们都十分欢迎。

## 一、贡献类型
1. 文档优化：修正文案、补充使用案例、完善平台指引
2. 代码迭代：修复BUG、优化脚本性能、拓展新功能
3. 流程补充：完善确权SOP，适配更多使用场景
4. 问题反馈：提交使用中遇到的故障、体验建议

## 二、提交流程
1. Fork 本仓库，基于 main 分支创建个人开发分支
2. 完成修改后，本地自测验证功能正常
3. 提交 Pull Request，并清晰描述修改内容、解决的问题
4. 维护者审核后合并代码

## 三、规范要求
1. 代码遵循 Python 基础编码规范，注释清晰易懂
2. 文档语言简洁通俗，保证新手可正常阅读理解
3. 所有修改不得移除项目原有确权核心逻辑与品牌标识
4. 请勿提交涉密、违规内容，共同维护项目合规性

## 四、问题反馈
使用过程中遇到问题，请新建 Issue，优先使用预设模板描述问题场景、复现步骤。
